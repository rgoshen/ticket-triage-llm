# Final Project Rubric – (100 Points)

I. Model & Inference (40 Points)

Model Functionality (20 Points):

Excellent (16-20 Points): Model successfully trained and running, producing meaningful outputs for the chosen task. Includes a LoRA if trained. Crucially, demonstrates a thoughtful evaluation process – creation and use of an evaluation dataset to assess model performance is encouraged.

Good (12-16 Points): Model runs with some functionality, but may have limitations or require further refinement. LoRA training is attempted but may not be fully successful. Attempts to evaluate the model, but the evaluation dataset may be limited or the process not fully thought through.

Fair (8-12 Points): Significant issues or doesn't produce reliable outputs. LoRA training is attempted but unsuccessful. Minimal or no attempt to evaluate the model.

Poor (0-8 Points): No functional model. No attempt to evaluate the model.

Innovation & Creativity (20 Points):

Excellent (16-20 Points): The project shows a definite stretch of the students skills and expertise. Demonstrates a clear understanding of where the field currently sits and how it can be improved.

Good (12-16 Points): Basic innovation is shown, but may not fully stretch the student's capabilities in this space. Creativity is shown, but mostly within the comfort zone.

Fair (8-12 Points): Limited or no creativity shown. Almost exclusively stayed in comfort zone.

Poor (0-8 Points): No innovation or creativity shown.

II. Production Environment & Programmability (30 Points)

Environment Setup (15 Points):

Excellent (12-15 Points): Model is successfully deployed in a chosen production environment (local, AWS, etc.). The environment is configured correctly and the model is accessible via an API endpoint.

Good (8-12 Points): Environment setup is partially successful, but may require some manual intervention or troubleshooting.

Fair (4-8 Points): Significant difficulties in setting up the production environment.

Poor (0-4 Points): Unable to set up a functional production environment.

Inference Pipeline (15 Points):

Excellent (12-15 Points): A robust and efficient inference pipeline is established, allowing for calls to the model. The pipeline is well-documented and optimized for performance. Includes appropriate sampling method.

Good (8-12 Points): Basic inference pipeline is implemented, but may have limitations or require further refinement. Sampling method is implemented but basic.

Fair (4-8 Points): Rudimentary inference pipeline with limited functionality.

Poor (0-4 Points): No functional inference pipeline.

III. Documentation & Presentation (30 Points)

Technical Documentation (15 Points):

Excellent (12-15 Points): Comprehensive documentation of the entire system – model, inference pipeline, API, deployment process, and sampling method.

Good (8-12 Points): Good documentation, but may have some gaps.

Fair (4-8 Points): Rudimentary documentation.

Poor (0-4 Points): Little to no documentation.

Demo & Presentation (15 Points):

Excellent (12-15 Points): Clear, concise, and engaging demonstration of the project.

Good (8-12 Points): Adequate demonstration with some explanation.

Fair (4-8 Points): Basic demonstration with limited explanation.

Poor (0-4 Points): Unclear or incomplete demonstration.
